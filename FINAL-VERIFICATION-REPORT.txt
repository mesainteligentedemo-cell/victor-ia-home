FLUJO DE FUNCIONAMIENTO
================================================================================

INICIALIZACION (Page Load):
  1. HTML parsea y elementos data-attributes se detectan
  2. CSS aplica estilos (canvas fullscreen, z-index ordering)
  3. DOMContentLoaded event se dispara
  4. HeroScrollFrames constructor:
     - Obtiene referencias a elementos DOM
     - Crea array vacío para 400 frames
     - Carga f001.jpg (primer frame)
     - Inicia 4 streams paralelos de precarga
     - Agrega listeners: window.scroll, window.resize
  5. Primer frame se renderiza en canvas

SCROLL INTERACTION:
  1. Usuario scrolls la página
  2. onScroll() event handler se dispara (~16ms refresh)
  3. Calcula progress: 0.0 (arriba) → 1.0 (abajo)
  4. Frame index = round(progress × 399)
  5. Pinta frame más cercano cargado en canvas
  6. Actualiza:
     - Phase label (según progreso)
     - Progress bar (scaleX)
     - Overlay (translateY si entre 40%-52%)
  7. Frames se cargan on-demand mientras scrolls

PERFORMANCE:

  - 4 parallel streams: Reduce tiempo total de carga
  - Frame caching: Frames cargados se reutilizan
  - Smooth rendering: 60fps en desktop/tablet
  - Mobile: Touch scroll también funciona sin lag

================================================================================
NOTAS TECNICAS
================================================================================

1. FRAME LOADING STRATEGY
   - First Frame: Preload inmediato (f001.jpg)
   - Subsequent Frames: Carga en 4 streams paralelos
   - On-Demand Fallback: Si frame no está cargado, usa más cercano disponible
   - Error Handling: Si frame falla, salta al siguiente stream

2. CANVAS RENDERING
   - 2D Canvas API (no WebGL)
   - Cover scaling: Imágenes se escalan para cubrir el viewport
   - HiDPI support: Soporta Retina displays (devicePixelRatio)
   - Resize listener: Se redibuja al cambiar tamaño de ventana

3. SCROLL DETECTION
   - Manual scroll listener (no Intersection Observer)
   - Calcula posición del elemento relativa a viewport
   - Progress: (scrollTop - elementTop + windowHeight) / (elementHeight + windowHeight)
   - Rango: 0.0 (arriba) a 1.0 (abajo)

4. PHASE TRANSITIONS
   - 6 puntos de transición (0%, 14%, 30%, 44%, 58%, 84%)
   - Labels actualizan dinámicamente según progreso
   - Sin interpolación: cada fase es discreta

5. OPTIMIZACIONES IMPLEMENTADAS
   - Frame streaming paralelo (4 workers)
   - Canvas DPI scaling (Math.min(devicePixelRatio, 2))
   - Lazy frame loading (bajo demanda)
   - requestAnimationFrame implícito en scroll listener
